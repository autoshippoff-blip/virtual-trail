# Architecture — AI Virtual Try-On SaaS Platform

> Plug-and-play Shopify widget. Fully multi-tenant. No Docker required.

---

## Platform overview

Two independent plug-and-play features share the same infrastructure:

| Feature | What it does | Widget trigger |
|---|---|---|
| **AI Virtual Try-On** | Customer uploads photo → AI renders them wearing the product | `TryOnWidget.init()` |
| **Size Intelligence** | Customer answers questions → AI recommends the right size | `SizeWidget.init()` (separate spec) |

A brand opts into one or both. Both features resolve the same tenant config, use the same auth, same database, same queue. Zero shared state between pipelines.

---

## Constraints and design decisions

| Constraint | Decision |
|---|---|
| No Docker on dev machine | All services run as managed cloud (Neon, Upstash, Cloudflare) or local Node processes |
| 2–3 day delivery | Minimal viable slice only — polling not WebSocket, no billing UI, no admin dashboard |
| AI agents (Antigravity) doing most code | Every module is a single-responsibility file with a typed contract at the top |
| Shopify integration | Public app with ScriptTag injection (no theme extension needed for MVP) |
| No OpenAI account | Compliments generated by Google Gemini Flash (free tier, no credit card for setup) |
| Minimize cost everywhere | Image compression, result caching, aggressive R2 cleanup, Upstash command budgeting |

---

## Cost strategy

Every AI call costs money. This table shows every cut made and why:

| Decision | Saving | Detail |
|---|---|---|
| Gemini Flash instead of OpenAI | ~100% on compliments | Free up to 1,500 req/day; $0.10/1M tokens after. At MVP scale: $0 |
| Compress user image before Segmind | 30–60% on Segmind payload | Resize to max 1024px, JPEG 85%. Smaller payload = faster = fewer timeouts = fewer retries |
| Cache compliment per product | Eliminates repeat Gemini calls | Same garment → same style note. Key: `compliment:{tenantId}:{productId}:{tone}`. TTL 24h |
| Delete user uploads after 24h | R2 storage | Selfies are not needed once job is done |
| Delete generated images after 7 days | R2 storage | Customers rarely return after a week |
| Widget polls every 3s not 2s | ~33% fewer Upstash commands | Stays inside 10k/day free tier longer |
| Railway Starter for both services | ~$5/mo fixed | API + Worker share one plan |

**Estimated cost per try-on:**

| Component | Cost |
|---|---|
| Segmind try-on API | ~$0.02–0.05 (model-dependent) |
| Gemini Flash compliment | ~$0.000003 (negligible) |
| R2 storage + egress | ~$0.0001 |
| **Total** | **~$0.02–0.05** |

Charge brands $0.10–0.20 per try-on → 2–5× margin from day one.

## Infrastructure — no Docker, all managed

| Layer | Service | Cost | Why |
|---|---|---|---|
| API + Worker runtime | Railway Starter | ~$5/mo | Git push → deploy, no Docker needed |
| Database | Neon (serverless Postgres) | Free | 0.5GB, Prisma-compatible |
| Redis / Queue | Upstash Redis | Free | 10k commands/day free tier |
| Object storage | Cloudflare R2 | Free | 10GB free, S3-compatible SDK |
| Widget CDN | Cloudflare Pages | Free | Global edge, auto-deploy from Git |
| Compliment AI | Google Gemini Flash | Free | 1,500 req/day free, no card required |
| Try-on AI | Segmind | Pay-per-call | Only variable cost — scales with usage |
| Error tracking | Sentry | Free | 5k errors/mo free tier |

**Total fixed cost: ~$5/month.** Only Segmind scales with usage.

**Local dev:** `pnpm dev` starts the API and worker as two Node processes. Neon and Upstash are accessed over the internet even locally.

## Database schema (Prisma)

```prisma
model Tenant {
  id            String         @id @default(uuid())
  name          String
  shopifyDomain String         @unique
  apiKey        String         @unique @default(uuid())
  features      String[]       // ["tryon", "size_intelligence"]
  createdAt     DateTime       @default(now())

  config        TenantConfig?
  products      Product[]
  tryonRequests TryonRequest[]
}

model TenantConfig {
  id             String   @id @default(uuid())
  tenantId       String   @unique
  tenant         Tenant   @relation(fields: [tenantId], references: [id])
  primaryColor   String   @default("#000000")
  complimentTone String   @default("friendly")  // friendly | luxury | playful
  logoUrl        String?
  segmindModel   String   @default("fashion-tryon-v1")
  updatedAt      DateTime @updatedAt
}

model Product {
  id               String         @id @default(uuid())
  tenantId         String
  tenant           Tenant         @relation(fields: [tenantId], references: [id])
  shopifyProductId String
  imageUrl         String
  category         String?
  createdAt        DateTime       @default(now())

  tryonRequests    TryonRequest[]

  @@unique([tenantId, shopifyProductId])
}

model TryonRequest {
  id                String    @id @default(uuid())
  tenantId          String
  tenant            Tenant    @relation(fields: [tenantId], references: [id])
  productId         String
  product           Product   @relation(fields: [productId], references: [id])
  status            String    @default("queued")  // queued | processing | completed | failed
  userImageKey      String?   // R2 key — deleted after 24h by cleanup processor
  generatedImageKey String?   // R2 key — deleted after 7 days by cleanup processor
  compliment        String?
  styleScore        Float?
  errorMessage      String?
  processingTimeMs  Int?
  complimentCached  Boolean   @default(false)  // true = served from Redis, no Gemini call made
  createdAt         DateTime  @default(now())
  updatedAt         DateTime  @updatedAt
}

model AuditLog {
  id        String   @id @default(uuid())
  tenantId  String
  action    String
  metadata  Json
  createdAt DateTime @default(now())
}
```

## AI processing pipeline

```
POST /v1/tryon
  │
  ├─ validate image (jpg/png/webp, <5mb, magic bytes check)
  ├─ compress image → max 1024px wide, JPEG 85% (sharp)
  ├─ resolve tenant + config (Redis cache 5min TTL)
  ├─ save TryonRequest (status=queued)
  ├─ upload compressed image → R2: {tenantId}/uploads/{requestId}
  ├─ enqueue job → BullMQ tryon-queue
  └─ return { jobId }

Worker picks up job
  │
  ├─ update TryonRequest status=processing
  ├─ get signed read URL for userImageKey from R2
  ├─ fetch product imageUrl from DB
  │
  ├─ [COST GATE] check compliment cache
  │     key: compliment:{tenantId}:{productId}:{tone}
  │     HIT  → use cached { compliment, score }, skip Gemini entirely
  │     MISS → flag for Gemini call after Segmind
  │
  ├─ call Segmind /v1/tryon
  │     { userImage: signedUrl, garmentImage: productImageUrl, model: config.segmindModel }
  │     timeout: 60s
  │
  ├─ upload generated image → R2: {tenantId}/generated/{requestId}
  │
  ├─ [if cache MISS] call Gemini Flash
  │     model: gemini-2.0-flash
  │     max_tokens: 80  (compliment is short — don't waste tokens)
  │     timeout: 10s
  │     on success → cache result TTL 86400 (24h)
  │     on failure → use fallback compliment, never fail the job
  │
  ├─ update TryonRequest:
  │     status=completed, generatedImageKey, compliment, styleScore,
  │     processingTimeMs, complimentCached
  │
  └─ cache job result → tryon:{jobId}:status TTL 180s

Cleanup processor (BullMQ cron, daily at 02:00):
  ├─ find TryonRequests where userImageKey IS NOT NULL and createdAt < now()-24h
  │     → delete R2 object, set userImageKey=null in DB
  └─ find TryonRequests where generatedImageKey IS NOT NULL and createdAt < now()-7days
        → delete R2 object, set generatedImageKey=null in DB

Retry strategy (BullMQ):
  attempts: 3
  backoff: exponential, 5000ms base
  On final failure: status=failed, errorMessage saved
```

## Gemini Flash — compliment generator

Get your API key from **aistudio.google.com** → "Get API key". Takes 2 minutes. No credit card. No billing setup. Free tier is 1,500 requests/day which covers hundreds of try-ons.

```typescript
// libs/ai/src/gemini.client.ts
import { GoogleGenerativeAI } from '@google/generative-ai';
import { config } from '../../../libs/config';

const TONE_GUIDE = {
  friendly: 'warm and encouraging',
  luxury:   'sophisticated and elevated',
  playful:  'fun and enthusiastic',
};

const SYSTEM = `You are a fashion stylist AI.
Only compliment the clothing, outfit, and styling choices.
Never mention body shape, weight, skin tone, attractiveness, race, or age.
Keep the compliment under 20 words.
Return ONLY valid JSON. No markdown. No explanation.`;

export async function generateCompliment(params: {
  tone: 'friendly' | 'luxury' | 'playful';
}): Promise<{ compliment: string; score: number }> {
  const genAI = new GoogleGenerativeAI(config.gemini.apiKey);
  const model = genAI.getGenerativeModel({ model: 'gemini-2.0-flash' });

  const prompt = `${SYSTEM}

Tone: ${TONE_GUIDE[params.tone]}
Return exactly: {"compliment": "...", "score": 8.5}
Score range: 6.0 to 9.9`;

  try {
    const result = await model.generateContent(prompt);
    const text = result.response.text().trim();
    const parsed = JSON.parse(text);
    return {
      compliment: String(parsed.compliment),
      score: Math.min(9.9, Math.max(6.0, Number(parsed.score))),
    };
  } catch {
    // Fallback — never crash a job over a compliment
    return { compliment: '✨ This outfit looks amazing on you!', score: 8.0 };
  }
}
```

## Image compression

Runs in the worker before calling Segmind. Reduces payload from ~2MB average to ~200kb.

```typescript
// libs/ai/src/image.utils.ts
import sharp from 'sharp';

export async function compressForTryOn(inputBuffer: Buffer): Promise<Buffer> {
  return sharp(inputBuffer)
    .resize({ width: 1024, height: 1024, fit: 'inside', withoutEnlargement: true })
    .jpeg({ quality: 85, mozjpeg: true })
    .toBuffer();
}
```

## Tenant isolation

Every DB query is scoped: `WHERE tenant_id = :tenantId`. The tenant is resolved from the request `tenantId` field, validated against the DB, cached in Redis for 5 minutes.

---

## Security

| Concern | Mitigation |
|---|---|
| Image abuse | Magic byte check, 5MB limit |
| AI cost abuse | 10 try-ons/IP/hour, 100/tenant/day |
| Key exposure | Segmind + Gemini keys — worker only, never in responses or logs |
| Generated image access | R2 signed URLs, 1-hour expiry |
| Shopify webhook spoofing | HMAC-SHA256 verification |

---

## Caching strategy

| Key | TTL | Purpose | Cost impact |
|---|---|---|---|
| `tenant:{id}:config` | 5 min | Tenant config | Saves DB reads |
| `tryon:{jobId}:status` | 3 min | Poll results | Saves DB reads |
| `compliment:{tenantId}:{productId}:{tone}` | 24 hr | Gemini result per garment | Eliminates repeat AI calls |
| `shopify:{domain}:token` | 24 hr | Shopify access token | Saves DB reads |
